# Standard: Required Reading

## Purpose
Reduce agent drift by making context consumption explicit.

## Reading tiers

### Universal
Files every major session should review:
- agents/README.md
- current workflow file
- project charter
- assumptions
- constraints
- decision log
- risk register

### Role-specific
Files required because of the current role:
- role file
- discipline-specific standards
- relevant reviews and handoffs

### Packet-specific
Files named directly in the prompt:
- story files
- ADRs
- design references
- implementation surfaces

## Reading rule
If a file is listed as required reading, the agent should not proceed as if it has read it unless it actually reviews it.

## Escalation
If required reading is missing or stale for critical work, document and escalate.
