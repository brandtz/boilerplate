# Workflow: Operating Model

## Summary
This repository uses a Master Agent plus specialist agents operating against shared markdown state.

## Universal rules
- No major work begins without required reading.
- Every cycle ends with a handoff.
- Review gates apply where defined.
- High-impact work requires multiple role reviews.
- Human sponsor decides major tradeoffs and approvals.

## Standard cycle
1. Enter phase
2. Read required context
3. Produce or update artifacts
4. Review
5. Rework if needed
6. Approve
7. Handoff to next phase

## Artifact hierarchy
1. Context files are the current project memory.
2. Workflow files define process.
3. Role files define identity and behavior.
4. Prompt files define task packets.
5. Handoffs and reviews record cycle outcomes.
