## Decisions Made
| Decision | Rationale |
|---|---|
| STATUS_THEME in `constants/statusTheme.ts` separate from STATUS_COLORS in chartConfig | Theme has Tailwind classes (bg/text) for badges; chartConfig has hex for chart.js; different consumers |
| useAccordion manages flat Set of IDs for both epic and story levels | Two separate instances avoid parent-child coupling; simpler state model |
| resolveLastUpdate accepts pre-computed taskIds for story/epic | Avoids re-traversing the epic tree; caller already has the data |
| FilterBar uses local state with callback props | Epics page owns the filtered list via useMemo; filter is ephemeral UI state |
| Completion formula: `done / (total - cancelled) × 100` | Per business rules Section 2.5; consistent with SummaryMetrics |
