# Prompts Directory

This directory contains all prompt files organized by lifecycle stage.

## Structure

```
prompts/
  index.md          ← canonical inventory (source of truth for dashboard)
  README.md         ← this file
  active/           ← prompts that are ready, in-progress, or blocked
  archive/          ← completed, superseded, or cancelled prompts
  templates/        ← reusable prompt authoring templates and examples
  generated/        ← prompts generated by the Master Agent before triage
  draft/            ← incomplete or unapproved prompts
```

## Lifecycle

1. **Draft** → A prompt is created or generated and placed in `draft/` or `generated/`.
2. **Ready** → After review, the prompt is moved to `active/` with `status: ready`.
3. **In Progress** → An agent begins executing the prompt. Status updated to `in_progress`.
4. **Done** → Execution completes. A session handoff is written. Status updated to `done`.
5. **Archived** → After handoff is verified and index updated, the prompt is moved to `archive/`. `archived_at` is set.
6. **Superseded** → If replaced, original is marked `superseded` with `superseded_by` and moved to `archive/`.
7. **Cancelled** → If no longer needed, marked `cancelled` and moved to `archive/`.

## Rules

- **`prompts/index.md` is the source of truth.** The dashboard reads this file first. Folder contents alone do not define project scope.
- **Archived prompts still count.** They contribute to project totals and completion metrics.
- **Never delete prompts.** Move them to `archive/` with a terminal status.
- **Never renumber existing prompts.** Use insertion numbering (e.g., `16.0.2`) instead.
- **Every done prompt must have a handoff.** The handoff path is recorded in the index and prompt frontmatter.
- **Downstream sync required.** When an inserted or superseding prompt changes dependencies, update all affected downstream prompts.

## Authoring a New Prompt

1. Copy `agents/templates/prompt-session-template.md` to `prompts/draft/` or `prompts/active/`.
2. Fill in all frontmatter fields.
3. Write the prompt body sections.
4. Add an entry to `prompts/index.md`.
5. If the prompt is ready for execution, place it in `prompts/active/`.

## Completing a Prompt

1. Execute the prompt.
2. Create a session handoff using `agents/templates/session-handoff-template.md`.
3. Update the prompt frontmatter: set `status: done`, `completed_at`, `session_handoff`.
4. Update `prompts/index.md` with completion details.
5. Commit and push all changes to the remote repository.
6. Move the prompt file to `prompts/archive/`.
7. Set `archived_at` in both the file frontmatter and the index.
