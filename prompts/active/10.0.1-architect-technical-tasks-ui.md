# Prompt 10.0.1: Architect — Technical Task Generation for UI (E2–E5)

## Sequence Position

- Prompt: 10.0.1
- Epic: E2, E3, E4, E5
- Story: All stories in E2–E5
- Tasks: All E2–E5 tasks
- Phase: Technical Task Generation

## Prerequisites

- 8.0.1 complete (Review gate approved)

## Context for the Agent

You are the Solution Architect. Break down all Epic E2–E5 stories into detailed technical tasks with implementation specifications. Each task should include component names, file paths, props/interfaces, state management, and test requirements.

## Required Reading

```text
agents/epics/project-manager-dashboard-epics.md

agents/decisions/ADR-003-ui-architecture.md
docs/dashboard-ux-wireframes.md
agents/context/architecture-overview.md
docs/test-strategy.md
```

## Scope

- For each E2–E5 task, specify: component file paths, props interfaces, state hooks, data flow, test cases
- Define the UI component folder structure
- Define page/route structure
- Define shared component contracts
- Produce technical task specifications as a markdown document

## Constraints

- Do not write implementation code — produce specifications
- Ensure consistency with wireframes and UI architecture ADR

## Validation Commands

```bash
test -f docs/technical-tasks-e2-e5-ui.md
```

## Handoff Instructions

Create handoff notes using `agents/templates/session-handoff-template.md`. Include:

- Component structure defined
- Any open design questions
