## Scope

- Implement refresh button that re-runs parser and updates DashboardState
- Add loading and error states during re-parse
- Implement file watcher using chokidar for local dev mode
- Add debounce (500ms) for rapid file changes
- Build repo selector UI component
- Implement repo capability detection (check for agents/epics/, prompts/, etc.)
- Wire repo switching to parser and state reload
- Implement local storage persistence for recent project paths
- Build recent-projects list in selector
- Write tests for refresh flow, capability detection, repo switching

## Constraints

- File paths in repo selector must be validated and sandboxed (no path traversal)
- File watcher must be optional and configurable
- Local storage uses only project paths, no sensitive data
