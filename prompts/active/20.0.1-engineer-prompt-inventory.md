# Prompt 20.0.1: Engineer — Prompt Inventory Table and Detail Drawer (E4-S1, E4-S2)

## Sequence Position

- Prompt: 20.0.1
- Epic: E4
- Story: E4-S1, E4-S2
- Tasks: E4-S1-T1 through T4, E4-S2-T1 through T4
- Phase: Implementation

## Prerequisites

- 19.0.1 complete (Epics view)

## Context for the Agent

You are the Senior Software Engineer. Build the prompt inventory table (sortable, filterable, with natural prompt-ID ordering) and the prompt detail drawer (metadata, raw prompt body rendered as markdown, prerequisites, required reading, session notes, changed files, handoff links).

## Required Reading

```text
docs/technical-tasks-e2-e5-ui.md
docs/dashboard-ux-wireframes.md
docs/project-manager-dashboard-data-contract.md
```

## Scope

- Build prompt inventory table with columns: number, title, epic, story, status, prerequisites, updated
- Implement column sorting with natural prompt-ID ordering (numeric tuple)
- Implement filtering by status, epic, and search text
- Build prompt detail drawer that slides in from right
- Render prompt body and handoff notes as markdown using react-markdown
- Display all metadata fields from prompt frontmatter
- Wire handoff links and changed files
- Write component tests for table sorting, filtering, drawer interaction

## Constraints

- Markdown rendering must sanitize content to prevent XSS
- Table must handle 250+ rows without performance degradation

## Validation Commands

```bash
cd apps/dashboard && npm test -- --testPathPattern="prompt-inventory|detail-drawer"
```

## Handoff Instructions

Create handoff notes. Include: components built, sort/filter implementation, markdown rendering approach.


## Session Close-Out Checklist

Before committing, the executing agent MUST complete ALL of the following steps.
This checklist is mandatory for both local and cloud agents.

1. **Create session handoff** using `agents/templates/session-handoff-template.md`
   - Save to `agents/handoffs/S-<date>-<seq>-<slug>.md`

2. **Update this prompt's frontmatter:**
   - Set `status: "done"`
   - Set `session_handoff` to the handoff file path created in step 1
   - Set `completed_at` to the current ISO-8601 timestamp
   - Set `updated_at` to the current ISO-8601 timestamp

3. **Update `prompts/index.md`:**
   - Change this prompt's row: status to `done`, add handoff path, set completed date, add summary note
   - Update the summary counts (decrement ready, increment done)

4. **Update `agents/context/status-dashboard.md`:**
   - Add this prompt to Active Work Packets as **DONE** with a brief note
   - Update Metrics counts (Prompts Completed, ready/done tallies)
   - Update Next Suggested Action to the next prompt in sequence
   - Update Awaiting Review if any items were resolved or added

5. **Update `agents/context/decision-log.md`:**
   - Add a row for each decision made during this session

6. **Commit and push:**
   ```bash
   git add -A
   git commit -m "Complete prompt <prompt_id>: <title>"
   git push
   ```
