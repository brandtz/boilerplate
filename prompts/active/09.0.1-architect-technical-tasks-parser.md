# Prompt 9.0.1: Architect — Technical Task Generation for Parser (E1)

## Sequence Position

- Prompt: 9.0.1
- Epic: E1 (Repo Data Contracts and Parsing Foundation)
- Story: E1-S1 through E1-S6
- Tasks: All E1 tasks
- Phase: Technical Task Generation

## Prerequisites

- 8.0.1 complete (Review gate approved)

## Context for the Agent

You are the Solution Architect. Break down all Epic E1 stories into detailed technical tasks with implementation specifications. Each task should include file paths, function signatures, interface definitions, and test requirements sufficient for an engineer to implement without ambiguity.

## Required Reading

```text
agents/epics/project-manager-dashboard-epics.md
agents/decisions/ADR-001-stack-selection.md
agents/decisions/ADR-002-parser-architecture.md
agents/context/architecture-overview.md
docs/test-strategy.md
```

## Scope

- For each E1 task, specify: file paths, function/module design, input/output contracts, error handling, test cases
- Define the parser module folder structure
- Define the TypeScript interfaces for all parsed models
- Define the CLI entry point specification
- Produce technical task specifications as a markdown document

## Constraints

- Do not write implementation code — produce specifications
- Focus on what each task must produce, not how to code it

## Validation Commands

```bash
test -f docs/technical-tasks-e1-parser.md
```

## Handoff Instructions

Create handoff notes using `agents/templates/session-handoff-template.md`. Include:
- Count of technical tasks specified
- Module structure defined
- Any open design questions
