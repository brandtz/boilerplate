### ADR-002 Conditions Resolution (Required)

The 8.0.1 review gate issued a GO with conditions: all 6 ADR-002 conditions **must** be resolved in this prompt before E1 implementation begins. The technical task document must explicitly address each:

1. **`prompts/index.md` as canonical source (ADR-002 §1):** Specify the two-phase parsing approach — (a) parse `prompts/index.md` for the canonical prompt registry, (b) scan `prompts/active/` and `prompts/archive/` for frontmatter details. Folder contents supplement but do not override the index inventory. Reference `docs/business-rules.md` Section 4 for file discovery order.

2. **`ParseWarning` interface (ADR-002 §2):** Define the `ParseWarning` interface that `DashboardState.warnings` references. Minimum shape: `{ file: string; line?: number; code: string; message: string; severity: 'error' | 'warning' | 'info' }`. Align warning codes with the error taxonomy in `docs/business-rules.md`.

3. **Missing `DashboardState` interfaces (ADR-002 §3, Risk R12):** Define `ProjectSummary`, `SummaryMetrics`, and `NextPromptInfo` interfaces referenced in `DashboardState`. Derive field specifications from the data contract and `docs/business-rules.md` counting/rollup rules.

4. **Prompt lifecycle fields (ADR-002 §4, Risk R10):** Add missing fields to the `ParsedPrompt` interface: `sessionHandoff`, `supersedes`, `supersededBy`, `insertReason`, `completedAt`, `archivedAt`. These are required for FR-6 (prompt inventory) and FR-7 (prompt detail drawer). Reference the data contract Section 1.1.

5. **Derived metrics computation (ADR-002 §5):** Document which metrics the Graph Builder layer computes (completion rollups, status counts, dependency resolution) vs. which are deferred to the UI layer. Reference `docs/business-rules.md` Section 2 for counting and rollup rules.

6. **Epic/story/task parsing schema (ADR-002 §6, Risk R11):** Define formal parsing rules for extracting epic → story → task hierarchy from `agents/epics/*.md` files. Reference `docs/business-rules.md` Section 4 for the combined markdown format specification.
