# Prompt 5.0.1: DevSecOps — Security Review of Architecture and Data Handling

## Sequence Position

- Prompt: 5.0.1
- Epic: E6 (Review, Quality, and Hardening)
- Story: E6-S4 (Security review for file/path handling)
- Tasks: E6-S4-T1 through E6-S4-T4

## Prerequisites

- 4.0.1 complete (Architecture review approved)

## Context for the Agent

You are the DevSecOps Engineer. Review the proposed architecture, parser design, and UI design for security vulnerabilities. Focus on file path handling, markdown rendering (XSS), repo selector (path traversal), and dependency supply chain.


## Scope

- Review file path handling for path traversal vulnerabilities
- Review markdown rendering approach for XSS vectors
- Review multi-repo selector for directory traversal risks
- Review dependency choices for known vulnerabilities
- Produce a security review findings document
- Update risk register with security-specific risks

## Constraints

- Do not implement fixes — produce findings and recommendations
- Do not block the project unless a critical vulnerability exists in the design

## Validation Commands

```bash
# Verify security review document was created
test -f docs/security-review-findings.md
```

## Handoff Instructions

Create handoff notes using `agents/templates/session-handoff-template.md`. Include:
- Security findings categorized by severity (critical, high, medium, low)
- Recommended mitigations for each finding
- Risk register updates made
