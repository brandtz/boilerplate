# Prompt 18.0.1: Engineer — Blockers Panel and Next Prompt Widget (E2-S3, E2-S4)

## Sequence Position

- Prompt: 18.0.1
- Epic: E2
- Story: E2-S3, E2-S4
- Tasks: E2-S3-T1 through T4, E2-S4-T1 through T4

## Prerequisites

- 17.0.1 complete (Summary cards and charts)

## Context for the Agent

You are the Senior Software Engineer. Build the blockers and warnings panel (blocked prompts, missing handoffs, invalid metadata) and the next prompt widget (full text viewer, copy-to-clipboard, rationale, prerequisite status).

## Required Reading

```text
docs/technical-tasks-e2-e5-ui.md
docs/dashboard-ux-wireframes.md
docs/project-manager-dashboard-data-contract.md
```

## Scope

- Build blockers panel: aggregate blocked prompts, missing handoffs, validation warnings from parser
- Build warnings panel: malformed frontmatter, unresolved prerequisites
- Build next-prompt widget: display prompt ID, title, rationale, prerequisite check
- Implement scrollable prompt text viewer in the widget
- Implement copy-to-clipboard for full prompt content
- Link to source file path
- Write interaction tests for copy action and prerequisite display

## Constraints

- Copy-to-clipboard must work in modern browsers without Flash or deprecated APIs
- Widget must display a clear message when no prompt is eligible

## Validation Commands

```bash
cd apps/dashboard && npm test -- --testPathPattern="blocker|next-prompt|widget"
```

## Handoff Instructions

Create handoff notes. Include: components built, copy-to-clipboard approach, test results.


## Session Close-Out Checklist

Before committing, the executing agent MUST complete ALL of the following steps.
This checklist is mandatory for both local and cloud agents.

1. **Create session handoff** using `agents/templates/session-handoff-template.md`
   - Save to `agents/handoffs/S-<date>-<seq>-<slug>.md`

2. **Update this prompt's frontmatter:**
   - Set `status: "done"`
   - Set `session_handoff` to the handoff file path created in step 1
   - Set `completed_at` to the current ISO-8601 timestamp
   - Set `updated_at` to the current ISO-8601 timestamp

3. **Update `prompts/index.md`:**
   - Change this prompt's row: status to `done`, add handoff path, set completed date, add summary note
   - Update the summary counts (decrement ready, increment done)

4. **Update `agents/context/status-dashboard.md`:**
   - Add this prompt to Active Work Packets as **DONE** with a brief note
   - Update Metrics counts (Prompts Completed, ready/done tallies)
   - Update Next Suggested Action to the next prompt in sequence
   - Update Awaiting Review if any items were resolved or added

5. **Update `agents/context/decision-log.md`:**
   - Add a row for each decision made during this session

6. **Commit and push:**
   ```bash
   git add -A
   git commit -m "Complete prompt <prompt_id>: <title>"
   git push
   ```
