# Prompt 26.0.1: UX — Accessibility Review and Keyboard Navigation (E6-S3)

## Sequence Position

- Prompt: 26.0.1
- Epic: E6
- Story: E6-S3
- Tasks: E6-S3-T1 through T4
- Phase: Validation

## Prerequisites

- 23.0.1 complete (UI code review approved)

## Context for the Agent

You are the Product Designer / UX agent. Audit the implemented dashboard for accessibility compliance. Check keyboard navigation, ARIA labels, color contrast, and screen reader compatibility.

## Required Reading

```text
agents/epics/project-manager-dashboard-epics.md
docs/dashboard-ux-wireframes.md
```

## Scope

- Audit all interactive elements for keyboard accessibility (tab order, focus indicators)
- Verify ARIA labels on dynamic content (drawers, expanding sections, charts)
- Test color contrast against WCAG 2.1 AA standards
- Test with screen reader on overview and prompt views
- Produce an accessibility audit report with findings and recommended fixes
- If fixes are needed, specify them as tasks for an inserted prompt (26.0.2)

## Constraints

- Do not implement fixes directly — produce an audit report
- Use inserted prompts for any remediation work

## Validation Commands

```bash
# Run axe accessibility linter if configured
cd apps/dashboard && npx axe-linter src/
```

## Handoff Instructions

Create handoff notes. Include: audit findings, WCAG compliance status, fix priorities.
