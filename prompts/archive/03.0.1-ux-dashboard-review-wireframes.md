# Prompt 3.0.1: UX Designer — Dashboard UX Review and Wireframes

## Sequence Position

- Prompt: 3.0.1
- Epic: E2 (Overview Dashboard), E3, E4 (cross-cutting)
- Story: Cross-cutting
- Tasks: N/A
- Phase: Product Definition

## Prerequisites

- 2.0.1 complete (BSA acceptance criteria)

## Context for the Agent

You are the Product Designer / UX agent. Review the PRD views, the UI architecture ADR, and the refined epics. Produce text-based wireframe descriptions for each of the five primary views. Identify usability concerns, navigation flow issues, and accessibility requirements.

## Required Reading

```text
docs/project-manager-dashboard-prd.md
agents/epics/project-manager-dashboard-epics.md
agents/decisions/ADR-003-ui-architecture.md
agents/context/product-brief.md
```

## Scope

- Produce text/ASCII wireframe descriptions for: Overview, Epics, Prompts, Sessions, Tasks views
- Define navigation flow between views
- Identify accessibility requirements (keyboard nav, ARIA, contrast)
- Flag any UX concerns with the proposed layout
- Document interaction patterns (drill-down, drawers, copy actions)

## Constraints

- Do not generate code or choose specific component libraries
- Text-based wireframes only (no image generation)
- Focus on information architecture and interaction patterns

## Validation Commands

```bash
# Verify wireframe document was created
test -f docs/dashboard-ux-wireframes.md
```

Create handoff notes using `agents/templates/session-handoff-template.md`. Include:
- List of views wireframed
- Key UX decisions and rationale
- Accessibility requirements identified
