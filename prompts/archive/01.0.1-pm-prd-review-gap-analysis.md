# Prompt 1.0.1: Product Manager — PRD Review and Gap Analysis

## Sequence Position

- Prompt: 1.0.1
- Epic: E1 (Repo Data Contracts and Parsing Foundation) + all epics
- Story: Cross-cutting
- Tasks: N/A

## Prerequisites

- None (first prompt in the sequence)

## Context for the Agent

You are the Product Manager. Review the PRD, data contract, and epics for the Project Manager Dashboard. Identify gaps, ambiguities, unstated assumptions, missing acceptance criteria, and conflicting requirements. Produce a structured gap analysis document.

## Required Reading

```text
docs/project-manager-dashboard-prd.md

agents/epics/project-manager-dashboard-epics.md
agents/context/project-charter.md

agents/context/assumptions.md
agents/context/constraints.md
```

## Scope

- Review all functional requirements (FR-1 through FR-16) for completeness
- Identify any user stories that lack clear acceptance criteria
- Flag requirements that conflict with constraints
- List any unstated assumptions
- Identify any missing non-functional requirements (performance, accessibility, etc.)
- Produce a structured gap analysis as a markdown document

## Constraints

- Do not rewrite the PRD — produce a review report
- Do not make stack or architecture decisions
- Do not generate code

## Validation Commands

```bash
# Verify gap analysis document was created
test -f docs/prd-gap-analysis.md
```
